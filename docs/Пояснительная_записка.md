## 1. Описание предметной области

«Бантуми» (Kalah) — стратегическая настольная игра из семейства Манкала, широко распространённая в Африке. Игра относится к классу игр с полной информацией и нулевой суммой, что делает её идеальным объектом для применения классических алгоритмов искусственного интеллекта.

### Правила игры

Доска состоит из 12 малых лунок (по 6 на каждой стороне) и 2 Калахов — хранилищ игроков. В начале партии в каждую лунку помещается N камней (по умолчанию 4). Игроки поочерёдно выбирают лунку на своей стороне, берут все камни и раскладывают их по одному против часовой стрелки:

- **Бонусный ход**: если последний камень попал в собственный Калах, игрок ходит снова
- **Захват**: если последний камень попал в пустую лунку своей стороны, игрок забирает его и все камни из противоположной лунки противника
- **Конец игры**: когда у одного из игроков заканчиваются все камни на его стороне

## 2. Архитектура приложения

Приложение построено по принципу **MVC (Model–View–Controller)**:

```
┌─────────────────────────────────────────┐
│              index.html (View)          │
│         Разметка игрового поля          │
└─────────────────┬───────────────────────┘
                  │ события DOM
┌─────────────────▼───────────────────────┐
│              ui.js (Controller)         │
│   Обработка кликов, анимации, журнал    │
└──────────┬──────────────────────────────┘
           │ вызовы методов
    ┌──────▼──────┐     ┌─────────────┐
    │  game.js    │     │   ai.js     │
    │   (Model)   │◄────│  (Strategy) │
    │ Игровая     │     │  Минимакс   │
    │ логика      │     │             │
    └─────────────┘     └─────────────┘
```

### 2.1 Модуль game.js — Игровая логика

**Класс `BantumiGame`** хранит полное состояние игры:

| Поле | Тип | Описание |
|------|-----|---------|
| `board` | `number[14]` | Массив: лунки 0–5 (P1), Калах 6, лунки 7–12 (P2), Калах 13 |
| `currentPlayer` | `1 \| 2` | Номер игрока, чья очередь |
| `gameOver` | `boolean` | Признак завершения партии |
| `winner` | `0 \| 1 \| 2 \| null` | 0 = ничья |
| `history` | `State[]` | Стек состояний для функции Undo |

**Ключевые алгоритмы:**

*Раскладывание камней:*
```
current = pitIndex
stones = board[pitIndex]; board[pitIndex] = 0
while stones > 0:
    current = (current + 1) mod 14
    if current == opponentKalah: continue
    board[current]++; stones--
```

*Противоположная лунка:* `opposite(i) = 12 - i` (работает симметрично для обеих сторон: 0↔12, 1↔11, ..., 5↔7)

### 2.2 Модуль ai.js — Компьютерный противник

**Класс `BantumiAI`** реализует алгоритм **Минимакс с альфа-бета отсечением**.

**Алгоритм Минимакс:**
- ИИ (Игрок 2) — максимизирующий игрок
- Человек (Игрок 1) — минимизирующий игрок
- Бонусный ход не переключает очерёдность в дереве поиска

**Эвристическая оценочная функция:**
```
score = Δkalah × 3 + ΔbonusMoves × 2 + Δcaptures × 1.5
```
где Δ — разность значений (ИИ минус человек)

**Сложность по глубине:**
| Уровень | Глубина | Примерное число узлов |
|---------|---------|----------------------|
| Лёгкий  | 2       | ~30                  |
| Средний | 5       | ~7 500               |
| Сложный | 9       | ~50 000 000          |

**Альфа-бета отсечение** сокращает реально просматриваемые узлы в 5–10 раз по сравнению с чистым Минимаксом.

### 2.3 Модуль ui.js — Пользовательский интерфейс

**Класс `BantumiUI`** управляет DOM:

- `_render()` — полная перерисовка доски после каждого хода
- `_onPitClick(idx)` — обработчик клика, вызывает `game.makeMove()` и при необходимости `ai.getBestMove()`
- `_scheduleAiMove()` — запускает ход ИИ через `setTimeout(400ms)` чтобы браузер успел обновить UI перед тяжёлым вычислением
- `_undo()` — восстанавливает предыдущее состояние из `game.history`

## 3. Обоснование технологического стека

| Выбор | Обоснование |
|-------|-------------|
| Vanilla JS (без фреймворков) | Отсутствие зависимостей, мгновенный запуск, простота изучения |
| CSS Grid + Flexbox | Точное позиционирование доски, адаптивность |
| ES6+ классы | Чёткое разделение на модули, поддержка в всех современных браузерах |
| Минимакс + α-β | Классический алгоритм для игр с полной информацией, хорошо изучен |
