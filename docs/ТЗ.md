## 1. Назначение и цели проекта

Целью проекта является разработка программной реализации настольной игры «Бантуми» (Kalah / Манкала) в виде клиент-серверного приложения. Бэкенд реализует игровую логику и предоставляет REST API. Фронтенд — десктопное приложение с графическим интерфейсом, взаимодействующее с бэкендом по HTTP.

Игра рассчитана на двух игроков, играющих на одном устройстве поочерёдно.

---

## 2. Требования к функциональности

### 2.1 Игровая логика (бэкенд)
| № | Требование |
|---|-----------|
| Ф-01 | Инициализация доски с N камнями в каждой из 12 лунок (N = 3–6) |
| Ф-02 | Раскладывание камней против часовой стрелки с пропуском Калаха противника |
| Ф-03 | Бонусный ход при попадании последнего камня в свой Калах |
| Ф-04 | Захват камней при попадании последнего камня в пустую лунку своей стороны |
| Ф-05 | Определение конца игры и автоматический сбор оставшихся камней |
| Ф-06 | Подсчёт итогового счёта и определение победителя (в т.ч. ничья) |
| Ф-07 | Отмена последнего хода (Undo) через стек состояний |

### 2.2 REST API (бэкенд)
| № | Метод | Эндпоинт | Описание |
|---|-------|----------|---------|
| А-01 | GET | `/api/v1/game/state` | Получить текущее состояние игры |
| А-02 | POST | `/api/v1/game/new?stones={n}` | Начать новую партию |
| А-03 | POST | `/api/v1/game/move/{pit}` | Сделать ход из лунки pit |
| А-04 | POST | `/api/v1/game/undo` | Отменить последний ход |

### 2.3 Графический интерфейс (фронтенд)
| № | Требование |
|---|-----------|
| Ф-08 | Отображение игровой доски: 12 лунок и 2 Калаха |
| Ф-09 | Количество камней в каждой лунке в реальном времени |
| Ф-10 | Подсветка последней лунки после хода |
| Ф-11 | Индикатор текущего хода (Игрок 1 / Игрок 2) |
| Ф-12 | Журнал ходов с отметкой бонусных ходов и захватов |
| Ф-13 | Диалог с результатом по окончании игры |
| Ф-14 | Кнопки «Новая игра» и «Отменить ход» |
| Ф-15 | Выбор количества камней в лунке (3–6) |

## 3. Нефункциональные требования

| Категория | Требование |
|-----------|-----------|
| Совместимость | macOS, Windows, Linux (Python 3.10+, Java 21+) |
| Производительность | Ответ API на любой запрос ≤ 200 мс |
| Надёжность | Некорректный ход возвращает `valid: false`, игра не ломается |
| Читаемость кода | Разделение на слои: controller / service / model (Java), отдельный класс UI (Python) |
| Документация | Комментарии к классам и методам на русском языке |
